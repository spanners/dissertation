In response to your question on Mathbin (is that visible to the public, by the way?)

Ignoring the Gammas for now, the rule says that if you have a command C which contains an undeclared variable iota, you can write

new iota in C

which is another command in which iota is a declared local variable.

[The Gammas just mean that both "C" and "new iota in C" can have other undeclared variables in them, and that there are the same ones in "C" that there are in "new iota in C".]

Think of these rules like this:

the parts on the left of the turnstile (the |- thing) are the undeclared variables

the part on the right is a piece of program text together with its type

the rule means "if you have a program like this (above the line) then you can write a program like this (below the line). They are just saying what programs are well-typed.

I hope that makes sense.
